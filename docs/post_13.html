RADI 2.4 is almost ready! I found out the proper configuration to run GZServer and GZClient with hardware acceleration. The final solution uses VirtualGL, TurboVNC and Nvidia drivers to utilize the GPU inside the container.

The performance measurements are (measured on Follow Line Exercise, with CPU intel i7 and GPU nvidia 1660ti):

  • Real Time Factor: from 0.93 to 0.98
  • CPU usage: from 70% to 45%
  • RAM usage: from 8.3GB to 7.3GB
  • GZClient FPS: from 22 to 47

Now, the application responds much faster and the RTF is close to 1. All the current exercises are having similar performance improvements.

We now have to test on other setups to confirm the results, and integrate this configuration with the previous one, as our users may or may not fullfill the requirements to run with hardware acceleration.
