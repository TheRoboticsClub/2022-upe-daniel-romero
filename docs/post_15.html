As stated in the previous post, Robotics Academy and Unibotics users with low computational power could experience connection errors due to some elements of the RADI not being able to initialize before the browser tries to connect with them. To avoid this errors, I worked on a controled startup. The browser now waits properly until it gets notice from the RADI that it is ready to connect to both websockets it uses.

This week I released a new RADI version (2.4.2) which included the aforementioned improvements and other user interface enhancements done by other colleagues.
